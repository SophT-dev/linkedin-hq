# zapier-webhook

This hook fires a Zapier catch webhook when Claude Code stops, passing the session context as the POST body. It is a general-purpose automation bridge: any Zap that starts with a Webhook trigger can be connected here to chain Claude Code output into the rest of your Zapier workflow ecosystem.

## When it fires

Fires on the `Stop` event by default. You can change the event in `hook.json` to `SessionEnd`, `PostToolUse`, or any other valid Claude Code hook event to match your automation's trigger requirements.

## What it runs

```
orchestrator/run.sh zapier fire
```

The script reads the session context from stdin and sends an HTTP POST to the configured Zapier webhook URL.

## Required environment variables

| Variable | Description |
|---|---|
| `ZAPIER_WEBHOOK_URL` | The catch webhook URL generated by Zapier for your Zap |

## Optional environment variables

| Variable | Description |
|---|---|
| `ZAPIER_PAYLOAD_FILTER` | Comma-separated list of session context keys to include in the payload (defaults to sending everything) |
| `ZAPIER_WEBHOOK_SECRET` | Shared secret added as an `X-Webhook-Secret` header for request validation on the Zapier side |
